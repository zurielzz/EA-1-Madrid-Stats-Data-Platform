# Reflection Report

MadridStats was the extracurricular technological activity I selected for this assignment. The project centers on a custom soccer analytics pipeline that collects, processes, and presents football data such as match statistics, player performance, standings, and recent form. I built the system myself from scratch as both a technical learning experience and a serious product direction that can be offered to customers who want automated soccer data systems for websites or other digital platforms. This aligns closely with my career goals in software engineering, sports analytics, and data systems.

From a technical perspective, I made decisions that prioritized modularity, maintainability, and reuse. I separated data collection, transformation, normalization, storage, and frontend presentation so the platform could grow without becoming tightly coupled. The data gathering and service layer are built in Python with FastAPI, the frontend application uses React and TypeScript, and the data is stored in a PostgreSQL relational database hosted on Neon. The deployed platform is hosted on Azure. I also improved the data pipeline, the frontend analytics experience, and the project documentation.

I worked independently on the technical side and used this project to strengthen both engineering depth and product thinking. The work is also relevant to an upcoming collaboration with a major sports and news outlet that has millions of followers across social media. Because of intellectual property and collaboration constraints, the production repository is private and cannot be shared publicly. For that reason, this public repository is limited to documentation, diagrams, screenshots, and participation evidence rather than source code.

In terms of quality, I believe the project shows strong alignment between my extracurricular work and my long-term professional direction. It demonstrates how I approach real software problems through data pipelines, structured system design, and practical frontend delivery. If I were repeating the activity, I would invest earlier in monitoring and testing. Those areas would improve reliability, make failures easier to trace, and strengthen the platform as it scales.
